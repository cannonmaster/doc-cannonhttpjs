To use the `get` method for making a GET request with CannonHttpJS, follow the steps below:

1. Make a GET request using the `get` method:

```javascript
import CannonHttpJS, {
  ResponseData,
  ExtendedRequestOptions,
} from "@cannonui/httpjs";

CannonHttpJS.setBaseUrl("http://localhost:3000");

CannonHttpJS.get("/get-endpoint", {})
  .then((response: ResponseData) => {
    // Handle the response
    console.log(response.data); // Access the response data
  })
  .catch((error: Error) => {
    // Handle any errors
    console.error(error);
  });
```

The `get` method accepts two parameters: `url` and `config`.

- The `url` parameter specifies the URL to which the GET request will be made.

- The example used base url to indicate a base url for every request. However, it is optional.

- The `config` parameter is an object that can contain additional request options. Since CannonHttpJS is baesd on fetch, you could use any valid fetch config option here.

Here is an example of how to use the `get` method with various features:

```javascript
const url = "https://api.example.com/endpoint"; // Replace with your API endpoint

// Optional configuration
const config: ExtendedRequestOptions = {
  params: { key: "value" }, // Optional URL parameters
  headers: { Authorization: "Bearer token" }, // Custom headers
  timeout: 5000, // Request timeout in milliseconds
};

CannonHttpJS.get(url, config)
  .then((response: ResponseData) => {
    // Handle the response
    console.log(response.data); // Access the response data
    console.log(response.status); // Access the response status code
    console.log(response.statusText); // Access the response status text
    console.log(response.headers); // Access the response headers
  })
  .catch((error: Error) => {
    // Handle any errors
    console.error(error);
  });
```

In the above example:

- The `params` property in the `config` object allows you to include URL parameters. For example, `{ params: { key: 'value' } }` will result in a URL like `https://api.example.com/endpoint?key=value`.

- The `headers` property in the `config` object allows you to set custom headers for the request. In the example, it sets the `Authorization` header to include a bearer token.

- The `timeout` property in the `config` object specifies the maximum time in milliseconds to wait for the request to complete. If the request takes longer than the specified timeout, an error will be thrown.

You can customize the behavior of the GET request by modifying the `config` object according to your requirements
